# 接口问题修复总结

## 一、已修复的实体类映射问题

### 1.1 DimIcPoints 实体类

**问题描述**: 
- 实体类缺少 `pointType`、`longitude`、`latitude`、`serviceHours`、`tel` 字段
- Mapper XML 中使用了这些字段，导致 MyBatis 反射错误

**修复内容**:
```java
// 新增字段
private String pointType;        // 网点类型
private Double longitude;        // 经度
private Double latitude;         // 纬度
private String serviceHours;     // 营业时间
private String tel;             // 联系电话

// 保留兼容字段
private String businessHours;   // 兼容旧版本
```

**影响接口**:
- `GET /bus/icpoints/list` - 查询 IC 卡网点列表
- `GET /bus/icpoints/all` - 获取所有网点基本信息
- `GET /bus/icpoints/count/byType` - 按类型统计网点
- `GET /bus/icpoints/gis/data` - GIS 地图展示

### 1.2 DimBusLanes 实体类

**问题描述**:
- 实体类字段与 Mapper XML 的 resultMap 不匹配
- XML 中使用了 `roadName`、`startPoint`、`endPoint` 等字段，但实体类不存在

**修复内容**:
```java
// 新增字段（与 XML 映射一致）
private String roadName;         // 道路名称
private String startPoint;       // 起点
private String endPoint;         // 终点
private String laneType;         // 专用道类型
private String direction;        // 行驶方向
private String timeLimit;        // 限行时间
private Double longitude;        // 经度
private Double latitude;         // 纬度

// 保留兼容字段
private Integer routeType;       // 兼容旧版本
private String routeName;        // 兼容旧版本
private String featureType;      // 兼容旧版本
private String featureCoords;    // 兼容旧版本
```

**影响接口**:
- `GET /bus/buslanes/list` - 查询专用道列表
- `GET /bus/buslanes/all` - 获取所有专用道
- `GET /bus/buslanes/count/byType` - 按类型统计
- `GET /bus/buslanes/coverage/byLine` - 线路覆盖分析
- `GET /bus/buslanes/gis/data` - GIS 地图展示

## 四、数据源配置说明

### 4.1 数据源注解

所有 Service 方法都使用了 `@DataSource(DataSourceType.SLAVE)` 注解，表示使用 ClickHouse 从库。

**配置位置**:
- 主数据源 (MASTER): MySQL - 系统业务数据
- 从数据源 (SLAVE): ClickHouse - 客流分析数据

**注意事项**:
1. 确保 ClickHouse 数据源已正确配置
2. 检查 `application.yml` 中的 ClickHouse 连接信息
3. 验证 ClickHouse 中是否存在对应的表和数据

### 4.2 ClickHouse 表清单

**维度表**:
- dim_lines - 公交线路表
- dim_stops - 公交站点表
- dim_schedules - 公交班次表
- dim_route_station_details - 线路站点详情表

- dim_ic_points - IC 卡服务网点表

**事实表**:
- fact_station_flow - 站点客流事实表

## 五、常见问题处理

### 5.1 MyBatis 反射错误

**错误示例**:
```
There is no getter for property named 'pointType' in 'class com.ruoyi.domain.DimIcPoints'
```

**解决方案**:
- 检查实体类字段是否与 Mapper XML 的 resultMap 一致
- 确保字段有对应的 getter/setter 方法（使用 Lombok 的@Data 注解）
- 检查字段名拼写是否正确

### 5.2 数据源切换错误

**错误示例**:
```
The data source 'SLAVE' does not exist
```

**解决方案**:
- 检查 DynamicDataSource 配置
- 确认 ClickHouse 数据源已初始化
- 检查 application.yml 中的数据源配置

### 5.3 SQL 语法错误

**错误示例**:
```
ClickHouse exception: Syntax error
```

**解决方案**:
- ClickHouse 的 SQL 语法与 MySQL 有差异
- 日期函数使用: `toDate()`, `toHour()`
- 类型转换使用: `CAST(xxx AS DECIMAL(10,2))`
- 空值处理使用: `NULLIF()`, `COALESCE()`
